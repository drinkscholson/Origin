# Origin v0.1.0 Release Notes

`v0.1.0` is the first public release of the Origin language prototype.

This release establishes the first complete Origin stack:

- a readable packet language
- frame compression
- a machine-native wire layer
- a hybrid frame-wire transport
- an AI onboarding path
- a deterministic compiler
- a corpus benchmark
- an FW1-first runtime prototype

## Release Scope

The goal of `v0.1.0` is not production deployment.

The goal is to prove that Origin can function as a real machine coordination stack rather than a purely narrative concept.

## Included Capabilities

### Core language

- `O1` packet schema
- packet codec, parsing, validation, and normalization
- evidence, confidence, intent, references, and context handling

### Workflow structure

- `F1` frame encoding for repeated traffic
- reference ledger
- derived conflict detection

### Machine-native transport

- `W1` lexicon-compressed wire bundle
- `FW1` adaptive hybrid transport
- automatic lexicon selection when a lexicon helps
- empty-lexicon fallback when frame sharing is already optimal

### Input and training

- controlled-English compiler
- compiler evaluation corpus
- AI quickstart, playbook, drills, and system prompt
- bootstrap runner for model-ready Origin teaching context

### Runtime proof path

- larger workflow corpus for transport evaluation
- FW1-first runtime prototype
- internal agent coordination that actually serializes through `FW1`

## Verified Results

From the bundled scripts at release time:

- sample packet benchmark: `49.2%` savings over English
- incident workflow `FW1`: `66.1%` savings over English
- corpus `FW1`: `61.1%` savings over English
- corpus `FW1`: `11.2%` savings over `F1`
- runtime internal `FW1`: `38.0%` savings over English
- compiler evaluation: `18/18 passed`

## What v0.1.0 proves

`v0.1.0` proves the following:

- Origin can express machine coordination as a real language structure.
- Origin can outperform English on compactness in repeated workflows.
- `FW1` can act as a real internal transport rather than a purely readable notation.
- English can remain at the edge while internal traffic shifts toward Origin-native transport.

## Known Limits

`v0.1.0` still has important limits:

- the runtime uses deterministic scripted agents rather than model-driven agents
- there is no binary transport yet
- controlled-English coverage is intentionally narrow
- the evaluation corpus is still small
- there is no hosted runtime or distributed network layer

## Next Step

`v0.1.1` should focus on tightening the stack:

- broader workflow coverage
- stronger runtime behaviors
- better packaging and release automation
- more explicit compatibility and upgrade rules
